This was an experiment of writing functional language but using completely different style of work.

I wanted to see how long can i work on a single project (I have a bad habit of restarting the same pet project over and over again).

Therefore I started writing a journal which you can see here: https://github.com/frondeus/project-s/blob/master/docs/journal/2025-05-14.md

I used this whitepaper https://www.cs.tufts.edu/~nr/cs257/archive/stephen-dolan/thesis.pdf for my inference theory.

This experiment lasted almost four months and was stopped by two reasons:
* I started experimenting with agentic coding which in august 2025 was not yet that good imho.
* I was going through chemiotheraphy (I wont that battle btw) and could no longer focus on anything for few months.

Overall i think it was a success.

What's nice about this project - is `test-runner/` crate which allowed me to have snapshot tests directly in my markdown file journal.
It meant, that I was able to interact with the language and debug it in a very structured manner.
Plus since im using Zed editor, i got syntax highlighting and LSP support for my codeblocks ;-) 

## Why project-s?

It's because back then i was quite impressed by https://janet-lang.org so this lang is also based on s-expressions. Bit LISPy but with static type inference.

## What's next?

Well, having `s-expression` based lang was fun idea but not very scalable in a long term. I learned a lot.
Found a new way of working on a bigger projects and I'll use that experience ... 
to start yet another functional language somewhere in the future ;-)
