Okay, so I used canonical form to produce types for native functions.
But there seems to be a problem with `list` function.

```
list
```

```type 
[Any] -> [Any]
```

When printing types we loose information about:
* Minimum and maximum length of the list
* Whether Any is a variable like ?T0 or not.

Room for improvement, eh?

```
(let :x (list 1 2 3))
x
```

```type 
[Number]
```

```
(let (:a :b :c) (list 1 2 3))
a
```

```type 
Number
```

Nested destruction:

```
(let (:a (:b :c)) (list 1 (list 2 3)))
b
```

```type
Error: Incompatible types
   ╭─[ <input>:1:1 ]
   │
 1 │ (let (:a (:b :c)) (list 1 (list 2 3)))
   │                    │    │               
   │                    ╰──────────────────── Expected tuple
   │                         │               
   │                         ╰─────────────── But got number
───╯

```

This doesnt work because

```
(list 1 (list 2 3))
```

Has a type of:
```type
[Number | [Number]]
```

What if we change the expression?

```
(let (:a (:b :c)) (list (list 2 3) (list 4 5)))

b
```

```type
Number
```

Yep, that works.
So the error message is bit confusing but in the end type error is correct.
List demands to have a single type across all elements.
