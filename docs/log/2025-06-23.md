The next thing i have to do is macro expansion.

So far macros are first class citizens and are executed at runtime.
It works but it makes type checking impossible to do.
We can't typecheck content generated by macros because we dont know what it is.

So, my LISP macros are basically syntax macros. You get AST, you return AST.
Therefore these macros are quite similar to `macro_rules!` in Rust.

Usually what you can do is to do macro expansion phase as a pass before type checking.
And i think this is what i will do.
