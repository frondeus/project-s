# Importing

Let's make importing working with type checker


```s
(let :foo (import "foo.s"))
foo
```


```foo.s
[1 2 3]
```


```type
[3 | 2 | 1]
```

```json
[
  1.0,
  2.0,
  3.0
]
```

Okay, I have an impression it might not work with let polymorphism of top level module

```s
(let :foo (import "foo.s"))
(foo 5)
foo
```

```foo.s
(let :f (fn (:x) x))
f
```

```type
(5) -> 5
```

Yeah, a function becomes monomorphic.

Okay so probably the best way of fixing it so do what Ocaml is doing.
This article: https://okmij.org/ftp/ML/generalization.html#levels
