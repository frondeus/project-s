# And and Or

There is one more thing. I'm not sure if i understand correctly what is going on here.
Based on MLSub whitepaper, OR operator can be only used in Positive polarity, and AND operator can be only used in Negative polarity.

What does it mean in practice?

```s
(let :x (if true "4" 5))
x
```

```type
"4" | 5
```

Yes. Now what if we have a function with that if?

```s
(let :x (fn (:a) (if a "4" 5)))
x
```

```type
(bool) -> "4" | 5
```

okay, but what if we use that function now?

```s
(let :x (fn (:a) (if a "4" 5)))
(x true)
```

```type
5 | "4"
```

Is that **REALLY** correct?
Aren't we missing something?

What if i have ID function

(let :id (fn (:x) x))

Is THAT correct?
