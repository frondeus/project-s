**Me:** What I was working on before the hospital was to make `Some` be both a type constructor and a runtime function at the same time. Just like in Rust. To change this syntax:

```s
(let :foo (: Option (enum :Some 1)))
```

To something more natural:

```s
(let :foo (Some 1))
```

And `Some` is simultaneously:
- An enum variant
- A function that takes one generic argument 'a and returns `Some('a)`

Or if the developer insists:

```s
(let :foo (: Option (Some 1)))
```

Because in this language enums are... "loose"

**Friend:** How loose? xD

**Me:** Here's an example that wouldn't compile in Rust:

```rust
enum Foo {
   A(usize)
}

enum Bar {
   A(usize)
}
```

In Rust, `Foo::A` is not the same as `Bar::A`. In my language it is - because after all, I have structural typing.

So theoretically, assuming I add treatment of enum variants as constructors, then `Some` in the context of being used as a function has the type:

```s
('a) -> enum { Some: ('a) }
```

I.e., I treat one enum variant as effectively an enum with one variant (xD)

**Friend:** Ah, I see yeah xD so partial enum

**Me:** And just like algebraically you can say that two types are structurally identical:

```s
{ foo: number } & { bar : number } == { foo: number, bar: number }
```

So enums too:

```s
enum { None: () } & enum { Some: ('a) } == enum { None: () | Some: ('a) }
```

But sometimes you might want to express that with:

```s
(let :foo (Some 1))
```

`foo` should have type `enum { None: () | Some: ('a) }` and not just `enum { Some: ('a) }`

Which by the way leads me to an additional puzzle I didn't think about. Because this example:

```s
(type :Option (enum
    :Some 'a
    :None ()
))
(let :foo (: Option (enum :Some 1)))
```

```eval
val foo : enum  {Some: (1) | None: ()} = {
  "Some": 1.0
}
- : () = []
```

It inferred to `Some: (1)` and not `Some(number)` xD
