## Implementing Super with Overlays

To support `super` like in Nix overlays, you need a `fix` function that can handle both self-reference and super (the previous/extended object). Here's how:

```
(let :fix (fn (:f) (do (let* :result (f result)) result )))

(let :fix-extend (fn (:ext :base) (do
    (let :base-inst (fix (fn (:self) (base self (obj/plain)))))
      
    (fix (fn (:self) (ext self base-inst)))
)))

(let :base-pkgs (fn (:self :super) 
    (obj/plain
        :a 1
        :b 2
        :c (thunk () (- (self :b) (self :a)))
    )
))

(let :extended-pkgs (fn (:self :super) 
    (obj/extend super
        :a 10
        :d (thunk () (- (self :a) (super :c)))
        :c 2
    )
))

(fix-extend extended-pkgs base-pkgs)
```

```type
{a: Number, b: Number, c: Number, a: Number, c: Number, d: Number}
```

```json
{
  "a": 10.0,
  "b": 2.0,
  "c": 2.0,
  "d": 9.0
}
```

This creates an overlay system where:
- `self` refers to the final result (with all overlays applied)
- `super` refers to the base/previous layer being extended
- You can override fields (like `:a`)
- You can add new fields (like `:d`)
- You can extend existing fields (like `:c` which adds 5 to the super's `:c`)

For multiple overlays, you can chain them:

```ignore 
(let :overlay1 (fn (:self :super)
    (obj/extend super :x 1)))

(let :overlay2 (fn (:self :super)
    (obj/extend super 
        :y 2
        :x (thunk () (+ (super :x) 10)))))

(fix-extend overlay2 (fix-extend overlay1 (obj/plain)))
```

This would create `{x: 11, y: 2}` where overlay2 extends overlay1's result.
