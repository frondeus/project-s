I think today is a day to add enums.

```s
(let :area (fn (:shape)
    (match shape
        (:circle :v)
           (* (v :radius) (v :radius) 3.14)
        (:rectangle :v)
           (* (v :width) (v :height))
    )
))
```

```eval
val area : forall (enum  {circle: ((:radius) -?-> number) | rectangle: ((:width) -?-> number ∧ (:height) -?-> number)}) → number = "<Function: LispFn>"
- : () = []
```

Okay, cool, I can create a function that takes an enum.
But how do I use it?

The problem is, currently we do not have type constructors, so I cannot just do:

```s
(Circle { :radius 5 })
```

That sucks a little bit but I think i will be able to make it happen in the future.

For now, tho, I think the best would be to add a new function called `enum`.

```s
(enum :circle { :radius 5 })
```

```eval
- : enum  {circle: ({radius: 5})} = {
  "circle": {
    "radius": 5.0
  }
}
```

Don't worry about runtime for now.

Let's see if type inference works correctly now:

```s
(let :area (fn (:shape)
    (match shape
        (:circle :v)
           (* (v :radius) (v :radius) 3.14)
        (:rectangle :v)
           (* (v :width) (v :height))
    )
))

(area (enum :circle { :radius 5 }))
```

```eval
val area : forall (enum  {circle: ((:radius) -?-> number) | rectangle: ((:width) -?-> number ∧ (:height) -?-> number)}) → number = "<Function: LispFn>"

```

Niiice.
