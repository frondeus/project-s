Oh okay, the problem is, that there is "implicit" scope
added by AST pass:

```s
(let :x 1)
5
```
is processed as

```s ignore
(do
   (let :x 1)
   #    x is scoped to this do expression
   5 # but 5 is returned from the do expression
)
```

I could theoretically add
```s ignore
(unscope
   (let :x 1)
   #    x is NOT scoped to this unscope expression
   5 # but 5 is returned from the unscope expression
)
```

but then we would have `:x` mixed with prelude.

And then we could do the hacky thing of have

```s ignore
(top-level
   (let :x 1)
   #    x is scoped to this top-level expression
   5
   # but leaving this top-level expression does not destroy the env.
)

```
